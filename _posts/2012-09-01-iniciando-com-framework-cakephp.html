Caso não saibam o que é um framework e as vantagens de seu uso sugiro que vejam esse post</a>.

Hoje irei demonstrar como iniciar uma aplicação com o CakePHP no Windows e dar os primeiros passos com essa ferramenta.

</a>

CakePHP é um framework web completo de código aberto (você pode contribuir se quiser!) escrito em PHP (como o nome diz) que foi iniciado em 2005.

Ele trabalha seguindo o padrão MVC (veja mais detalhes sobre MVC aqui</a>) e é voltado para o desenvolvimento rápido. Possui uma excelente documentação e uma comunidade muito ativa.

Download e Instalação</h2>

Para começar nossa primeira aplicação, precisamos acessar o site oficial (http://cakephp.org/</a>) do CakePHP e clicar no link "Download" à direita para baixar a última versão estável (no meu caso é a 2.2.1).

</a>

Após o download, copie o arquivo compactado baixado, cole na pasta do servidor onde PHP pode ser rodado (C:\wamp\www\</code> caso esteja usando Wamp Server), extraia e renomei para "first-project". Feito isso você deverá ver a seguinte estrutura:

</a>

- app: essa pasta é onde você irá trabalhar, nela serão armazenadas toda a lógica, imagens, arquivos de estilos (CSS) e comportamentos (Javascript) do site.

- lib: pasta que contém o núcleo (conhecido como "core") do CakePHP, nela estão armazenadas as principais classes e componentes do framework. Nenhum arquivo deve ser alterado aqui pois caso você atualize a versão do Cake todas alterações serão perdidas.

- plugins: diretório responsável por armazenar plugins (mini aplicações do CakePHP) de terceiros.

- vendors: pasta responsável por armazenar bibliotecas e classes externas para serem utilizadas no projeto, como por exemplo uma classe para redimensionamento de imagens.

- .htaccess: arquivo de configuração do servidor Apache responsável por fazer reescrita de urls (usando mod_rewrite</code> do Apache que deve estar ativo). Ele encaminha todas requisições feitas nesse diretório para app/webroot/</code>. Se você acessar http://localhost/first-project/some_file.php</code> no navegador por exemplo, o Cake irá procurar o arquivo some_file.php</code> dentro da pasta app/webroot/</code> invés de app/</code>.

- index.php: arquivo responsável por definir algumas constantes que são usadas pelo framework.

Agora que já conhecemos os elementos principais da estrutura do Cake podemos acessar nosso projeto pela primeira vez e ver o que temos, para isso digite no browser o endereço de desenvolvimento (geralmente http://localhost</code>) e clique na pasta do projeto.

</a>

Essa é a tela inicial onde o CakePHP faz algumas verificações.. Observe as mensagens em vermelho, elas estão dizendo que devemos alterar o valor da Security.salt</code> e da Security.cipherSeed</code> localizadas no arquivo de configuração core.php</code>. Vamos fazer o que ele está pedindo, acesse a pasta app/Config/</code> com seu editor favorito e veja o que temos.

</a>

Os arquivos que mais nos interessam por enquanto são:

- core.php: arquivo responsável por armazenar principais configurações do projeto como: nível de debug, cache, logs, sessões etc. Abra o arquivo e substitua o valor atual da Security.salt</code> e Security.cipherSeed</code> para qualquer string randômica (aperte qualquer letra e número), esses campos servem para serem incluídos nas criptografias geradas pelo Cake, deixando mais difícil a decodificação (+1 para segurança aqui).

- database.php.default: arquivo responsável por armazenar as configurações de acesso ao banco de dados. Aproveitando que estamos aqui, crie um banco de dados no MySQL ou outro que quiser (o CakePHP atualmente suporta MySQL, PostgreSQL, SQLite e SQL Server), abra o arquivo database.php.default</code>, digite o endereço do banco em host</code>, usuário em login</code>, senha em password</code>, nome do banco em database</code> e o qual fonte de dados você usará em datasource</code> (você encontra uma lista desse último no próprio arquivo), salve e renomei o arquivo para database.php</code> para ativar a conexão com banco de dados.

- email.php.default: arquivo responsável por armazenar configurações para envio de emails através da classe CakeEmail</code>.

- routes.php: arquivo responsável por definir as rotas de nossa aplicação.

Após fazermos as primeiras configurações já temos o CakePHP pronto para uso como na imagem abaixo.

</a>

Rotas, Requisições e Layouts</h2>

O Cake já vem com algumas configurações para funcionar, mas geralmente não usamos ao desenvolver nossos projetos, então devemos fazer nossas próprias.

Layouts</h3>

Para começar, temos o layout. Mas o que é layout no Cake? Layout nada mais é do que o HTML da sua aplicação que não muda entre todas as páginas, nele definimos normalmente o cabeçalho, menu principal, rodapé e aplicamos as folhas de estilos (CSS) e arquivos Javascript usados em todo o site. A função do layout é facilitar a manutenção, permitindo que tenhamos apenas um arquivo de estrutura básico para manter.

Vamos criar nosso primeiro layout, para isso, acesse a pasta app/View/Layouts</code>, abra o arquivo default.ctp</code>, remova o conteúdo dele e copie o abaixo.

[code language="html+php" linenos="1" hl_lines="9 15 19"]




<br /> Primeira Aplicação com CakePHP<br /> </title><br /> <?php<br /> echo $this->fetch('css');<br /> ?><br /> </head><br /> <body></p> <div id="container"> <div id="content"> <?php echo $this->fetch('content'); ?><br /> </div><br /> </div><br /> <?php<br /> echo $this->fetch('script');<br /> ?><br /> </body><br /> </html><br /> [/code]</p> <p>Observe as linhas destacadas, temos algo importante aqui. A linha 9 diz para o Cake onde os arquivos CSS devem ser inclusos, a linha 15 diz onde será incluído o conteúdo que muda de cada página (content) e a 19 diz onde os arquivos Javascript ficarão. Em um layout, devemos ter pelo menos a chamada para o bloco content (linha 15).</p> <p>Acesse <code>app/View/</code> e exclua a pasta <code>Pages</code> e vá até <code>app/Controller/</code> e exclua o arquivo <code>PagesController.php</code>. Se você acessar o site agora deverá ver uma mensagem de erro, calma que já iremos acertar..</p> <h3>Rotas e Requisições</h3></p> <p>O Cake já vem com um conjunto de rotas configuradas que funcionam muito bem. Primeiro você define o controller que deseja acessar, depois a action e em seguida passa os parâmetros se tiver. Veja um exemplo:</p> <p>http://meusite.com/produtos/todos</p> <p>Se nenhum rota criada por nós estiver definida, nosso framework irá procurar por um controller chamado <code>ProdutosController</code> na pasta <code>app/Controller/</code> e dentro desse controlador ele irá procurar uma action (que nada mais é do que um método) com o nome de <code>todos</code>, como abaixo.</p> <p>[code language="php" linenos="1"]<br /> <?php<br /> class ProdutosController extends AppController{<br /> function todos(){</p> <p> }<br /> }<br /> ?><br /> [/code]</p> <p>Observe que nosso controller possui o nome dele concatenado com o sufixo <code>Controller</code> e herda (extends) do controlador <code>AppController</code>, essa é uma convenção do Cake e devemos sempre criar um controlador seguindo ela. Se você criar o arquivo acima e acessar o endereço dele verá que teremos uma mensagem de erro dizendo <code>Missing View</code>, isso nos diz que temos que criar uma view com o mesmo nome da action definida no controller. Nesse caso crie um arquivo chamado <code>todos.ctp</code> (cake template - ctp) dentro da pasta <code>app/View/Produtos/</code> e verá que o erro sumirá. Todas as views do controlador <code>produtos</code> devem ficar nessa pasta tendo o mesmo nome da action com a extensão <code>ctp</code>.</p> <p>Se tivermos um controller chamado <code>Categorias</code> como abaixo por exemplo, o Cake irá procurar as views <code>listagem.ctp</code>, <code>cadastro.ctp</code> e <code>index.ctp</code> dentro da pasta <code>app/View/Categorias/</code> quando acessarmos os respectivos endereços <code>http://meusite.com/categorias/listagem</code>, <code>http://meusite.com/categorias/cadastro</code>, <code>http://meusite.com/categorias</code>. Detalhe, quando não definimos uma action como no caso da última url, o Cake irá entender que é para chamar a <code>index</code>, faça o teste!</p> <p>[code language="php" linenos="1"]<br /> <?php<br /> class CategoriasController extends AppController{<br /> function listagem(){</p> <p> }</p> <p> function cadastro(){</p> <p> }</p> <p> function index(){</p> <p> }<br /> }<br /> ?><br /> [/code]</p> <p>Agora vamos definir algumas rotas, abra o arquivo <code>routes.php</code> na pasta <code>app/Config/</code> e observe que temos duas rotas definidas.</p> <p>[code language="php" linenos="1"]<br /> <?php<br /> Router::connect('/', array('controller' => 'pages', 'action' => 'display', 'home'));</p> <p>Router::connect('/pages/*', array('controller' => 'pages', 'action' => 'display'));<br /> ?><br /> [/code]</p> <p>Remova essas linhas e deixe apenas a linha abaixo:</p> <p>[code language="php" linenos="1"]<br /> <?php<br /> Router::connect('/', array('controller' => 'home', 'action' => 'index'));<br /> ?><br /> [/code]</p> <p>Aqui nós definimos a rota raiz do site, caso acessemos <code>http://meusite.com</code>, o Cake chamará a action <code>index</code> do controller <code>Home</code>. Como ainda não criamos ele, vamos fazer agora, crie um arquivo com nome <code>HomeController.php</code> dentro da pasta <code>app/Controller/</code>, defina nossa action <code>index</code> e crie a pasta <code>Home</code> dentro de <code>app/View/</code> e nela crie nossa view <code>index.ctp</code>, veja a relação abaixo.</p> <p><a href="http://blog.glaucocustodio.com/wp-content/uploads/2012/09/conveções-do-cakephp.png"><img class="alignnone size-full wp-image-411" title="conveções-do-cakephp" src="http://blog.glaucocustodio.com/wp-content/uploads/2012/09/conveções-do-cakephp.png" alt="" width="670" height="593" /></a></p> <p>Agora abra o arquivo <code>app/View/Home/index.ctp</code>, digite <code>Olá mundo</code>, rode no navegador e veja nossa primeira aplicação no Cake funcionando.</p> <p>Lembra desse código <code><?php echo $this->fetch('content'); ?></code> que temos dentro do layout? Então, todo o conteúdo das views serão apresentados no lugar onde ele está definido. Se você olhar o código fonte da raiz do site verá que nosso <code>Olá mundo</code> está onde pedimos para ele estar, ou seja, dentro da div com id <code>content</code>.</p> <h2>Concluindo..</h2></p> <p>Hoje vimos como instalar, fazer as primeiras configurações e rodar nossa aplicação com CakePHP.
