Fala galera, beleza?

Hoje estou aqui para ensinar como criar nosso primeiro projeto</strong> com o framework Ruby on Rails</strong>. Pra quem não sabe o que é Ruby on Rails pode conhecer um pouco mais aqui</a>.

Se você já conhece mas não tem instalado em sua máquina, veja esse post</a> onde explico como preparar um ambiente Rails no Linux. Se você usa Windows, instale uma máquina virtual ou crie dual boot pois Rails roda melhor em sistemas unix-like (Unix, Linux, OS X).

</a>

Iniciando projeto</h2>
Primeiro de tudo vamos criar um projeto Rails, para isso abra o terminal, acesse a pasta onde ficará o projeto e digite o comando rails new nome-do-projeto -d=mysql</code> (substitua "nome-do-projeto" pelo nome que você deseja dar).

</a>

O argumento -d=mysql</code> informa que o projeto usará banco de dados MySQL invés do SQLite que é padrão.</div>

O Rails irá criar uma pasta com o nome do projeto (nesse caso first-project</code>) dentro do diretório atual (pwd</code>). Vamos acessar a pasta e iniciar o servidor para testar a aplicação. Use o comando rails s</code> ou rails server</code> para iniciar o servidor.

</a>

Observe que ao tentar inicar o servidor obtemos uma mensagem de erro dizendo que não foi possível encontrar um "manipulador" de Javascript. Esse erro ocorre nessa versão do Rails (3.2.8), para resolver, precisamos acessar o arquivo Gemfile</code> que está dentro da pasta do projeto e adicionar a gema therubyracer</code> através do comando gem therubyracer</code> (linha 24), veja como ficará.

</a>

Gems são como bibliotecas em nossos projetos, elas são códigos Ruby que podemos portar, muito efeciente. Veja mais em http://rubygems.org/</a>.</div>

O arquivo Gemfile</code> diz ao Rails quais gems iremos usar, mas depois de definirmos precisamos baixar elas para nossa máquina, para isso execute o comando bundle install</code> na pasta do projeto. Agora inicie o servidor (rails s</code>) e veja que não teremos mais o erro, você deverá ver o WEBrick</code> (servidor padrão do Rails) rodando como abaixo.

</a>

Pressione Ctrl+C</code> para desligar o servidor.</div>

Configurando banco de dados</h2>
Dissemos ao Rails que usaremos banco de dados MySQL, mas precisamos dizer ainda qual a base de dados (database</code>), usuário (username</code>) e senha (password</code>). Abra o arquivo config/database.yml</code> e nas configurações de desenvolvimento defina os dados necessários.

</a>

Observe que temos configurações de banco de dados para três ambientes: desenvolvimento, teste e produção. Por enquanto trabalharemos apenas com desenvolvimento (development</code>).</div>

Precisamos criar o banco no MySQL ainda, para isso execute o comando rake db:create</code> no terminal.

</a>

Com isso já temos nosso projeto no ponto para começarmos a desenvolver, agora inicie o servidor novamente, abra o navegador e acesse http://0.0.0.0:3000/</code> para ver a tela de boas vindas do Rails.

</a>

Essa é a tela padrão de boas vindas que se encontra em public/index.html</code>.</div>

Pare o servidor (Ctrl+C</code>) e exclua essa página (rm public/index.html</code>) pois não precisaremos mais dela (ela só serve para dizer que o Rails está rodando ok).

Definindo rota raiz</h2>
Vamos dizer ao Rails qual é a nossa rota raiz, abra o arquivo config/routes.rb</code> e descomente (remova o #</code> da frente) a linha root :to => 'welcome#index'</code> e substitua welcome</code> por home</code> ficando assim.

</a>

Agora toda vez que acessarmos a url raiz de nosso site (http://localhost:3000</code>) seremos mapeados para o controller home</code> e action index</code>.

Olá mundo</h2>
Se acessarmos nosso projeto pelo navegador agora teremos problemas pois o Rails não irá encontrar nosso controller e action definidos no routes.rb</code>, vamos criar eles. Digite rails g controller home index</code> no terminal.

</a>

Esse rails g</code> vem de rails generate</code>, podemos usar ele para criar controllers, models, views etc.. Para saber mais digite rails g --help</code>.</div>

Se você olhar na pasta app/controllers/</code> e app/views/home/</code> verá que temos nosso controller e view criados em seus respectivos locais.

</a>

Para finalizar, vamos criar uma variável de instância na action do controller e exibir na view, abra o app/controllers/home_controller.rb</code> e adicione o comando @welcome = 'sejam bem vindos'</code> dentro da action index</code>.

</a>

Toda variável de instância começa com @</code> no Ruby e elas sempre são enviadas para a view no Rails, portanto se criarmos qualquer variável com @</code> no controller ela estará disponível na view.

Abra a view app/views/home/index.html.erb</code> e vamos exibir o valor definido no controller junto com uma mensagem de olá, veja.

</a>

<%=</code> inicia a tag do Ruby e já exibe o que estiver na frente (como se fosse do PHP). Para código que não precisa de output usa-se apenas <%</code>.</div>

Feito isso inicie o servidor e acesse o projeto no navegador em http://localhost:3000</code> e já veremos nosso olá mundo</code>.

</a>

Conclusão</h2>
Por enquanto é só pessoal, hoje aprendemos:

  • criar um projeto Rails</li>
  • conceito de gems</li>
  • definir rota raiz</li>
  • gerar controller através do rails g</code></li>
  • variáveis de instância</li>
  • exibir conteúdo nas views</li>
    </ul>
