E ai pessoal, tudo bom?

Venho falar de Django</strong> hoje, irei mostrar como preparar um ambiente de desenvolvimento no Linux</strong> (Ubuntu 12.04 LTS), explicar como essa ferramenta funciona e criar um projeto na prática. Se você não conhece o Django, recomendo que leia esse post</a> onde faço uma breve introdução.

</a>

Índice</h2>

  • Instalar ambiente de desenvolvimento Django 1.4 no Linux (Ubuntu)</a></li>
  • Aprender como Django trabalha</a></li>
  • Criar um site (projeto)</a></li>
  • Configurar um projeto</a></li>
  • Criar uma aplicação plugável (sistema de notícias)</a></li>
  • Gerenciar registros pela área administrativa</a></li>
  • Listar registros</a>
    • Urls</a></li>
    • Views</a></li>
    • Templates</a></li>
      </ul>
      </li>
      </ul>

      Instalação</h2>
      Os passos abaixo são destinados para Linux Ubuntu (12.04 LTS), caso você esteja no Windows e não queira trocar (o que não é recomendado), pode ver como preparar um ambiente de desenvolvimento       aqui</a>.

      Django é um framework escrito em Python, por isso não precisamos nos preoupar com a linguagem no Linux porque ela já vem instalada.

      Como irei utilizar banco de dados MySQL e o gerenciador de pacotes PIP nesse tutorial, preciso instalá-los, rode o comando sudo apt-get install python-mysqldb python-pip</code> e tudo deve estar ok.

      Instale também as bibliotecas libmysqlclient-dev</code> e python-dev</code> através do comando sudo apt-get install libmysqlclient-dev python-dev</code>.

      Vamos baixar o Django agora, rode o comando wget https://www.djangoproject.com/m/releases/1.4/Django-1.4.3.tar.gz</code> no terminal onde deseja salvar o arquivo ou entre no site</a>.

      Extraia o arquivo:
      [code language="bash"]tar xzvf Django-1.4.3.tar.gz[/code]
      Acesse a pasta:
      [code language="bash"]cd Django-1.4.3[/code]
      E instale:
      [code language="bash"]sudo python setup.py install[/code]
      Aguarde um instante e já teremos nosso ambiente Django para brincar!

      Entendendo o Django</h2>
      O Django é um framework MVC, ou seja, divide em três camadas nosso código, porém ele utiliza outro nome para o que conhecemos por model, view e controller. Com ele falamos em MVT, que é model, view e template. O que é isso?

      • Model é onde a lógica acontece, todo regra de negócio fica aqui.</li>
      • View é onde fazemos a intermediação entre a camada de lógica (model) e apresentação (template). Ela faz o mesmo papel dos controllers do Rails e CakePHP.</li>
      • Template é a camada de apresentação, deve ser usada lógica mínima aqui, o Django também não nos permite fazermos muito além do que devemos. Essa camada faz o papel das views do Rails e CakePHP.</li>
        </ul>
        A forma de trabalho do Django também é um pouco diferente do que muitos podem estar acostumados, com ele falamos em aplicações plugáveis. Esse conceito serve para aumentar a produtividade e evitar a repetição de código (don't repeat yourself).

        Imagina que trabalhamos em uma empresa que faz sites e que em quase todos eles temos uma galeria de fotos. Após já termos uma galeria pronta, o que você faria quando chegasse um novo serviço? Faria outra do início ou copiaria e adaptava uma já existente?

        No Django, essa galeria de fotos seria uma aplicação plugável, assim toda vez que chegasse um novo serviço apenas teriamos que "plugar".

        Cada novo site que chegasse na empresa seria um projeto Django, que nada mais é do que um conjunto de aplicações plugáveis e configurações específicas.

        Assim temos a lógica: um projeto pode possuir muitas aplicações e uma aplicação pode estar em muitos projetos.

        Criando um Projeto</h2>
        Acesse via terminal o local que deseja criar o projeto e execute o comando django-admin.py startproject project_name</code>, substitua "project_name" pelo nome do seu projeto.

        </a>

        Será criado um diretório com o nome do projeto, ao acessar e listar o conteúdo veremos que temos outro diretório com o mesmo nome, o primeiro de fora não é importante, podemos dar o nome que quisermos, mas o interno será usado no projeto e deve-se tomar cuidado ao mudar.

        </a>

        Nesse ponto já podemos rodar o projeto, para isso execute o comando python manage.py runserver</code> na pasta do projeto.

        </a>

        O Django validará os models (ainda não temos nenhum) e iniciará o servidor de desenvolvimento na porta 8000 por padrão, ao acessar o endereço 127.0.0.1:8000</code> obteremos a tela inicial.

        Podemos definir a porta que queremos rodar, basta colocar o número na frente. Ex: python manage.py runserver 8080</code> (para porta "8080").</div>
        </a>

        Esse servidor é apenas para desenvolvimento e não deve ser usado em produção, para isso escolha outro como Apache, Ngix etc..</div>

        Configurando o Projeto</h2>
        Abra o arquivo settings.py</code> que está dentro da pasta do projeto e vamos fazer algumas configurações.

        Defina os dados de administrador do site:
        [code language="python"]ADMINS = (
        ('Admin', 'admin@provedor.com'),
        )[/code]
        Acerte o time zone e o language code para sua localidade:
        [code language="python"]TIME_ZONE = 'America/Sao_Paulo'
        LANGUAGE_CODE = 'pt-br'[/code]
        Vamos configurar o banco de dados, defina os dados da conexão como abaixo para utilização do MySQL (adeque os dados de acesso para sua máquina):
        [code language="python"]DATABASES = {
        'default': {
        'ENGINE': 'django.db.backends.mysql',
        'NAME': 'first_django_project',
        'USER': 'root',
        'PASSWORD': '',
        'HOST': 'localhost',
        'PORT': '',
        }
        }[/code]

        Não esqueça de criar um banco de dados com o nome definido acima, no meu caso, irei criar um chamado first_django_project</code>.
        </div>

        Usando South para Migrar Banco de Dados</h3>
        O Django possui um comando chamado syncdb</code>, toda vez que criamos uma aplicação com um model, precisamos rodar o comando python manage.py syncdb</code> para que o Django varra nosso model e crie uma tabela com os campos definidos.

        Isso é maravilhoso, único parte ruim, é que quando alteramos o model - implementamos um novo campo por exemplo, ele não espelha essa alteração para nós no banco de dados, então precisamos editar a tabela manualmente.

        Para resolver isso, foi criado o South, ele é um migrador inteligente de dados e banco de dados para projetos Django.

        Primeiro, vamos instalar ele em nossa máquina através do comando:
        [code language="bash"]pip install south[/code]

        Rode como super usuário ou com sudo</code> caso obtenha erro.</div>
        Agora precisamos adicionar ele em nosso projeto. Lembra que eu disse lá em cima que um projeto Django é constituído de várias aplicações plugáveis? Então, agora precisamos "plugar" o South em nosso projeto, para isso adicione south</code> em INSTALLED_APPS</code> dentro do arquivo settings.py</code>. Toda vez que precisarmos plugar uma aplicação em nosso projeto, deve-se adicionar o nome dela nessa tupla no arquivo de configuração.

        Agora vamos rodar o comando responsável por criar as tabelas de nossa aplicação, execute:
        [code language="bash"]python manage.py syncdb[/code]
        O South utiliza algumas tabelas que ainda não estão criadas, por isso precisamos rodar o comando syncdb</code>, mas será só dessa vez.

        Veja o retorno do comando:

        </a>

        Como temos o sistema de autenticação django.contrib.auth</code> que já vem por padrão nas aplicações instaladas, quando rodamos pela primeira vez o syncdb</code>, o Django pergunta se queremos criar um usuário, é recomendado que já seja criado agora, isso será usado na área administrativa de nosso exemplo mais pra frente.

        Quanto ao South, ainda precisamos definir algumas coisas mais, assim que criarmos nossa primeira aplicação volto a falar dele..

        Criando uma Aplicação Plugável</h2>
        Vamos criar nossa primeira aplicação Django agora, para isso execute o comando python manage.py startapp news</code> no diretório do projeto, "news" é o nome de nossa aplicação.

        Será criado uma pasta com o nome da aplicação contendo os seguintes arquivos:

        </a>

        • __init__.py: arquivo em branco que diz ao Python que esse diretório deve ser considerado como um Python package (veja mais aqui</a>);</li>
        • models.py: esse arquivo conterá os campos e as principais regras de negócio de nossa aplicação. Geralmente uma aplicação pode ter mais de um model por arquivo - diferentemente do CakePHP e Ruby on Rails, por isso é chamado "models";</li>
        • views.py: arquivo que fará a intemediação entre os models e os templates;</li>
        • tests.py: como o nome já diz, onde serão realizados os testes.</li>
          </ul>
          Agora que criamos nossa aplicação, vamos plugar ela no projeto, lembra? Adicione a string news</code> no INSTALLED_APPS</code> do arquivo settings.py</code>.

          Com nossa aplicação plugada, vamos criar nosso primeiro model então, abra o arquivo models.py</code> e adicione o código:
          [code language="python"]# -*- coding: utf-8 -*-

          from django.db import models

          class News(models.Model):
          title = models.CharField('Título', max_length=100)
          publication_date = models.DateTimeField('Data de publicação', auto_now_add=True)
          content = models.TextField('Conteúdo')

          class Meta:
          verbose_name = 'notícia'
          verbose_name_plural = 'notícias'

          def __unicode__(self):
          return self.title[/code]

          O trecho # -*- coding: utf-8 -*-</code> serve para informar qua estamos utilizando a codificação UTF-8.</div>
          Vamos analisar o que temos aqui..

          Começamos criando uma classe News</code> que herda de models.Model</code> para termos as funcionalidades de um model Django.

          Depois definimos três campos para nossa aplicação, um título de tamanho máximo de 100 caracteres, um campo de data que é preenchido automaticamente (auto_now_add=True</code>) e outro para o conteúdo da notícia que é do tipo texto. Outros tipos de campos podem ser vistos aqui</a> na documentação oficial do Django.

          A classe Meta</code> serve para definir algumas informações sobre o model, mais pra frente veremos seu uso.

          O método __unicode__</code> define o retorno padrão, nesse caso definimos o título.

          Lembra do South? Então, sempre quando queremos automatizar uma aplicação para que as alterações nos models sejam espelhadas no banco de dados, precisamos criar uma migração inicial, ela nada mais é do que um arquivo que poussi a estrutura de nossos models. Para criar a migração inicial rode o comando python manage.py schemamigration news --initial</code> substituindo "news" pelo nome de sua aplicação.

          </a>

          Depois de criada a migração inicial, vamos rodar ela agora para que as tabelas necessárias possam ser criadas no banco de dados, execute o comando python manage.py migrate news</code>.

          </a>

          Daqui pra frente, sempre quando alterarmos os models da aplicação "news", basta executarmos python manage.py schemamigration news --auto</code> para gerar a migração e depois python manage.py migrate news</code> para que as alterações sejam efetuadas no banco de dados, assim não precisamos ficar nos preocupando com campos de tabelas e focamos no desenvolvimento.

          Podemos usar o comando python manage.py migrate --list</code> para exibir todas migrações definidas e se elas já foram aplicadas ou não (lendo a tabela south_migrationhistory).</div>

          Interface Administrativa</h2>
          Uma das melhores features do Django sem dúvida é a interface administrativa, com ela podemos rapidamente construir funcionalidades rotineiras que nos tomariam mais tempo e código utilizando outros ferramentas.

          Ela já vem pronta e tudo o que precisamos fazer é plugar, para isso basta remover o comentário da linha 'django.contrib.admin'</code> no INSTALLED_APPS</code> de nosso já velho conhecido arquivo settings.py</code>.
          [code language="python"]INSTALLED_APPS = (
          'django.contrib.auth',
          'django.contrib.contenttypes',
          'django.contrib.sessions',
          'django.contrib.sites',
          'django.contrib.messages',
          'django.contrib.staticfiles',
          'south',
          'news',
          # Uncomment the next line to enable the admin:
          'django.contrib.admin',
          # Uncomment the next line to enable admin documentation:
          # 'django.contrib.admindocs',
          )[/code]
          Caso você queira, também podem ativar a documentação da área administrativa removendo o comentário da linha 'django.contrib.admindocs'</code>, mas por enquanto não irei fazer isso.

          A área administrativa já está ativa, só precisamos habilitar o acesso a ela, basta abrirmos o arquivo urls.py</code> dentro da pasta do projeto e remover o comentário das linhas pertinentes ao admin. Ficaremos com algo assim:
          [code language="python"]from django.conf.urls import patterns, include, url

          # Uncomment the next two lines to enable the admin:
          from django.contrib import admin
          admin.autodiscover()

          Listando Registros</h2>

          Urls</h3>
          Para fazer a listagem dos registros precisamos definir qual url será utilizada, abra o arquivo urls.py</code> e adicione a linha abaixo:
          [code language="python"]url(r'^$', 'news.views.index', name='home'),[/code]

          O mapeamento de urls no Django é feito com expressão regular.</div>
          Aqui criamos uma url de nome home (name='home'</code>) que mapeia o endereço raiz do site (através da expressão regular ^$</code>) para a view index</code> que está no arquivo views.py</code> da aplicação news</code>.

          Views</h3>
          Vamos criar a view responsável por exibir os registros, abra o arquivo views.py</code> de nossa aplicação e adicione o seguinte código:
          [code language="python"]from django.shortcuts import render_to_response
          from news.models import News

          def index(request):
          return render_to_response('news/index.html',{
          'news': News.objects.all().order_by('-publication_date')
          })[/code]
          Nesse trecho fazemos as importações necessárias, criamos a view index</code> que renderiza o template news/index.html</code> e passamos a variável news</code> contendo todas as notícias cadastradas ordenadas de forma decrescente (o -</code> faz o papel de DESC</code> no SQL) pela data de publicação.

          O Django nos fornece uma API para realizarmos operações com o banco de dados pelos models, no exemplo acima, utilizamos o método all</code> para obter todos os registros, veja mais aqui</a>.</div>
