Fala galera, hoje estou aqui para falar de uma ferramenta que conheci há pouco tempo e que achei bem interessante, a gem Serve</a>.

Você já precisou criar um protótipo ou um pequeno site para trabalho de escola, faculdade ou até mesmo no serviço? Se já, então esse post é pra você.

</a>

Ano passado mesmo, um professor da faculdade pediu para que cada grupo da sala fizesse um site puro em HTML de até cinco páginas.

Imagina a frustação ter que copiar uma página pronta, colar e renomear para começar a fazer outra. Sem contar quando precisa mudar algo no layout (header e footer por exemplo), lá vamos nós então abrindo HTML por HTML para alterar.

Pra quem está acostumado trabalhar com linguagens dinâmicas isso é um trabalho chato e muito desgastante.

Por isso venho apresentar o Serve</strong>, segundo a própria documentação oficial, ele é um pequeno servidor web baseado em Rack e um framework de prototipagem rápida para aplicações web</em> (especificamente Ruby on Rails)</em>.

Com ele podemos criar e trabalhar em um projeto usando tecnologias como Sass, Compass, CoffeeScript, Markdown, Textile, Haml, ERB, Slim etc, além de recursos como views, layouts e partials.

Depois de finalizar o projeto, você pode facilmente exportar para HTML e CSS. Assim você agiliza o desenvolvimento e minimiza seu trabalho.

Instalação e Estrutura</h2>
Como o Serve é um gem, tudo o que precisamos fazer para instalar é rodar o comando gem install serve</code> no terminal.

Vamos criar um projeto agora, acesse a pasta onde ele ficará e rode o comando serve create nome-do-projeto</code>.

</a>

Vamos analisar a estrutura criada:

  • public: onde deve ser armazenado todos arquivos públicos, como imagens, Javascript e CSS.</li>
  • views: onde são armazenadas as views, layout, helpers e partials. Aqui ficam os ERB, Haml, Slim etc..</li>
  • stylesheets: local que contém os arquivos Sass e Scss a serem compilados para CSS.</li>
  • Gemfile: arquivo que lista as dependências do projeto, aqui são incluidas as gems utilizadas.</li>
  • config.ru: arquivo de configuração do servidor Rack.</li>
    </ul>

    Executando</h2>
    Vamos rodar nossa aplicação, dentro da pasta do projeto, execute bundle install</code> para ver se precisa instalar alguma dependência e depois rode serve</code> para executar o servidor na porta 4000 (padrão).

    </a>

    Tudo certo, agora é só brincar com o framework e criar um projeto. Eu fiz um pequeno site utilizando Haml como template engine.

    </a>

    Deixo a dica aqui para criar links, basta você colocar no href</code> da tag a</code> o nome da view sem a extensão, por exemplo, criei uma view com nome de about.html.haml</code>, no link, faço a chamada para /about</code>.

    Exportando</h2>
    Agora vem a melhor parte, hora de exportar o projeto Serve para HTML.

    É muito simples, basta você executar o comando serve export nome-do-projeto pasta-de-saida</code>, substituindo lógicamente "nome-do-projeto" pelo caminho da aplicação Serve e "pasta-de-saida" pelo caminho onde você deseja exportar.

    Feito isso, teremos nosso protótipo limpo e sem esforços desnecessários.

    </a>
