Fala pessoal! Segue abaixo uma lista com os principais comandos de git usados no dia-a-dia, com o passar do tempo vou adicionando mais..

Geral</h2>
Adicionar todos arquivos alterados no stage (deixar pronto para serem comitados):
[code language="bash"]git add .[/code]
Adicionar arquivos trackeados excluídos no stage:
[code language="bash"]git add -u[/code]
Remover arquivos do stage (prontos para commit) que já são trackeados:
[code language="bash"]git reset HEAD nome_arquivo[/code]
Remover arquivos do stage que ainda não são trackeados:
[code language="bash"]git rm --cached nome_arquivo[/code]
Refazer um commit quando necessário:
[code language="bash"]git commit -m "mensagem" --amend[/code]
Exibir alterações feitas depois do último commit que ainda não foram stageadas:
[code language="bash"]git diff[/code]
Exibir alterações feitas depois do último commit que já foram stageadas:
[code language="bash"]git diff --cached[/code]
Excluir um commit local (substitua o número abaixo pelo seu commit)
[code language="bash"]git reset --hard 7d01b96b7fd1a1e5bfa1971b50c51e55efa75646[/code]
Forçar o git a excluir um commit no repositório remoto, primeiro exclua localmente e depois rode:
[code language="bash"]git push origin HEAD --force[/code]
Abrir interface gráfica (gitk):
[code language="bash"]gitk[/code]
Ver número de commits por usuário:
[code language="bash"]git shortlog -s[/code]

Stash</h2>
Jogar alterações para o stash:
[code language="bash"]git stash[/code]
Ver itens no stash:
[code language="bash"]git stash list[/code]
Aplicar o último item adicionado no stash:
[code language="bash"]git stash apply[/code]
Aplicar um stash específico (liste todos os stashs e substitua 'numero_do_stash' pelo número desejado):
[code language="bash"]git stash apply stash@{numero_do_stash}[/code]
Limpar stash:
[code language="bash"]git stash clear[/code]

Branches</h2>
Criar um branch:
[code language="bash"]git branch nome_branch[/code]
Criar um branch e dar checkout (mudar pra ele):
[code language="bash"]git checkout -b nome_branch[/code]
Criar um branch "trackeando" (fazendo acompanhar) um branch remoto:
[code language="bash"]git checkout -b nome_branch_local origin/nome_branch_remoto[/code]
Mudar de branch:
[code language="bash"]git checkout nome_branch[/code]
Excluir um branch local:
[code language="bash"]git branch -d nome_branch[/code]
Excluir um branch remoto:
[code language="bash"]git push origin --delete nome_branch[/code]
Exibir todos os branches locais:
[code]git branch[/code]
Exibir todos os branches locais e remotos:
[code language="bash"]git branch -a [/code]
Fazer o git enxergar todos branches remotos:
[code language="bash"]git remote update[/code]
Ver origens remotas definadas para o branch atual:
[code language="bash"]git remote show [/code]
Excluir origem remota para o branch atual:
[code language="bash"]git remote rm nome-origin[/code]
Adicionar origem remota para o branch atual:
[code language="bash"]git remote add nome-origin git@xxxbucket.com:username/nome-repositorio.git[/code]
Trazer ("mergear") alterações de outro branch para o branch atual:
[code language="bash"]git merge nome-branch-a-ser-baixado [/code]
Ver apontamentos de branches do git, na pasta do .git, rode:
[code language="bash"]cat .git/config [/code]

Configuração</h2>
Definir configuração de nome global:
[code language="bash"]git config --global user.name "Seu nome" [/code]
Definir configuração de email global:
[code language="bash"]git config --global user.email "email@dominio.com.br"[/code]
Colorizar saídas:
[code language="bash"]git config --global color.ui auto [/code]
Criar alias (cria um alias "st" para "status", agora pode-se usar git st</code>):
[code language="bash"]git config --global alias.st 'status' [/code]
Definir editor padrão do git (utilizado em merges etc)
[code language="bash"]git config --global core.editor "vim"[/code]
