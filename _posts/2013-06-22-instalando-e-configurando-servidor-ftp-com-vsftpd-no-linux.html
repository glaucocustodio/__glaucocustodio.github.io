Fala pessoal, hoje venho ensinar como instalar e configurar um servidor FTP no Linux</strong>, mais precisamente no Ubuntu 12.04.

Dentre as muitas opções de programas que temos por ai, irei instalar o vsftpd</strong></a>, que parece ser muito seguro e é utilizado pela RedHat e OpenSuse além de outras empresas..

Entendendo o FTP</h2>
Por padrão o protocolo de transferência de arquivos utiliza duas portas para comunicação, a 21 para comandos e 20 para transferência dos dados.

Porém nem sempre a porta 20 é utilizada para transferência de dados, isso depende do modo que ele está configurado: ativou ou passivo.

Modo Ativo</h3>
No modo ativo, o cliente conecta-se através de uma porta randômica maior que 1023 (N > 1023) na porta 21 do servidor. Então o cliente começa ouvir na porta N + 1 e envia para o servidor essa porta, o servidor conecta-se de volta na porta especificada pelo cliente (N + 1) através da porta local 20.

Representação:

</a>

O problema com esse modo do FTP é que o servidor que começa uma conexão para transferência de dados, isso pode gerar problemas com o firewall do cliente pois está recebendo uma solicitação de fora e que geralmente é bloqueada.

Modo Passivo</h3>
Para resolver essa questão, foi desenvolvido o modo passivo (PASV). Nesse modo, o cliente abre duas portas randômicas maiores do que 1023 (N > 1023 e N+1). A primeira porta conecta-se com a porta 21 do servidor, mas invés de informar o servidor a porta para comunicação de dados, o cliente usará o comando PASV. Como resultado, o servidor vai abrir uma porta randômica (ou não, se configurarmos) P e então enviará essa porta P como resposta ao PASV para o cliente. Por fim, o cliente inicia o conexão através da porta N+1 na porta P do servidor para transferência de dados.

Representação:

</a>

Observe que no modo passivo não utilizamos a porta 20.</div>
Essa foi uma breve abordagem sobre o funcionamento do FTP e dos modos ativo e passivo, recomendo esse excelente link</a> caso você queira saber mais.

Instalação e Configuração</h2>
Agora que já temos uma base, vamos à instalação.

Primeiro, atualize a lista de pacotes e depois instale o vsftpd:
[code language="bash"]sudo apt-get update[/code]
[code language="bash"]sudo apt-get install vsftpd[/code]
Após a instalação, podemos testar acessando o servidor FTP localmente como anônimo (usuário sem senha), basta rodarmos o comando ftp localhost</code> e digitarmos "anonymous" no usuário e deixar a senha vazia.
</a>

Veja que estamos recebendo a mensagem "Consider usign PASV" porque estamos no modo ativo.</div>
Com nosso servidor instalado e funcionando, vamos alterar algumas configurações no arquivo que se encontra geralmente em /etc/vsftpd.conf</code>.

Vamos começar desabilitando acesso anônimo de nosso servidor. Abra o arquivo de configuração para editar e defina anonymous_enable=NO</code>.
[code language="bash"]sudo vim /etc/vsftpd.conf[/code]
</a>
Se tentarmos acessar o servidor agora não conseguiremos pois estamos sem nenhuma forma de autenticação.

Toda vez que alterarmos o arquivo de configuração precisamos reiniciar o servidor FTP, use o comando sudo service vsftpd restart</code> para isso.</div>
Precisamos de usuário para acessar nosso servidor, vamos dizer ao vsftpd para permitir login com usuários locais (do sistema operacional). Abra o arquivo de configuração e descomente a linha local_enable=YES</code>.

</a>

Uma boa prática é criar um grupo para usuários de FTP no Linux, assim facilita a administração de permissões para esses indivíduos. Execute o comando abaixo para criar um grupo chamado "ftpusers".
[code language="bash"]sudo groupadd ftpusers[/code]
Já temos o grupo, vamos criar um usuário chamado "ftpuser", adicionar ele no grupo "ftpusers" e definir uma senha:
[code language="bash"]useradd -g ftpusers ftpuser[/code]
[code language="bash"]passwd ftpuser[/code]
Se tentarmos acessar nosso servidor FTP agora, veremos que precisamos criar uma pasta com o mesmo nome do usuário dentro de /home</code>. Vamos criar esse diretório do usuário, mudar o owner para "ftpuser" e o grupo para "ftpusers". Também precisamos remover o acesso de escrita desse diretório, pois o vsftpd não permite que a pasta do usuário (somente ela, os subdiretórios podem ter) tenha acesso de escrita:
[code language="bash"]cd /home[/code]
[code language="bash"]sudo mkdir ftpuser[/code]
[code language="bash"]sudo chown ftpuser:ftpusers ftpuser[/code]
[code language="bash"]sudo chmod a-w ftpuser[/code]
Após as operações acimas, devemos ter isso:
</a>

Como o diretório do usuário não tem acesso de escrita, iremos criar um diretório chamado public</code> dentro de /home/ftpuser</code> onde o usuário poderá armazenar seus arquivos e depois mudaremos o owner para o usuário e seu grupo (para que ele consiga ter acesso total)
[code language="bash"]sudo mkdir /home/ftpuser/public[/code]
[code language="bash"]sudo chown ftpuser:ftpusers /home/ftpuser/public[/code]
A partir de agora já conseguimos acessar o servidor FTP com o usuário criado, só que tem algo de errado ai, o usuário consegue acessar qualquer diretório do sistema a partir do diretório dele, basta executar cd /</code> para acessar a raiz do Linux por exemplo..

Descomente a linha chroot_local_user=YES</code> para enjaular o usuário apenas no diretório dele, assim ele só poderá acessar subdiretórios de sua pasta.
</a>

Até agora está tudo certo, porém ainda estamos utilizando o modo ativo, isso pode nos gerar problemas.. Vamos ativar o modo passivo, para isso precisamos definir pasv_enable=YES</code> no arquivo de configuração.
</a>

Nesse servidor não foi necessário, mas em alguns pode ser necessário definir o ip do servidor FTP no qual o cliente deve fazer a solicitação, se for seu caso, basta definir pasv_address=XXX.XXX.XXX.XXX</code> (troque XXX.XXX.XXX.XXX pelo ip do seu servidor FTP) no arquivo de configuração.

Pode-se também definir uma faixa de portas na qual o servidor FTP irá usar para transferir arquivos, basta definir pasv_min_port=9040</code> e pasv_max_port=9050</code> no arquivo de configuração, nesse caso estariamos liberando as portas 9040 até 9050 para transferência no modo passivo.

Se você desejar pode mudar onde será o diretório inicial do usuário FTP (o padrão é /home/nome_usuario</code>), basta definir local_root=/var/$USER</code> no arquivo de configuração, caso você utilize a variável $USER</code> do sistema no caminho (como nesse exemplo), defina também a linha user_sub_token=$USER</code> para o vsftpd entender como uma variável. No exemplo dado, ao acessarmos o servidor FTP com o usuário "ftpuser" o vsftpd estaria procurando o diretório /var/ftpuser</code>.

Não esqueça que o diretório inicial deve ser do usuário e grupo desejados e a raiz não pode ter permissão de escrita.</div>

Para finalizar, muitas vezes podemos precisar que o diretório inicial do servidor FTP aponte para um outro diretório que não seja um subdiretório (que esteja fora da home), a melhor forma para fazer isso seria montar o diretório desejado dentro da pasta inicial do usuário. Temos aqui um exemplo: sudo mount --bind /var/webapps/meu-blog /home/usuario_ftp/blog</code>, assim cada vez que acessarmos o diretório "blog" do usuário estaremos na verdade acessando o "meu-blog" dentro de "var".
